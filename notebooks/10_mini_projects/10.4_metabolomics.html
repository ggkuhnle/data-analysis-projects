Load or Simulate a Metabolomics Dataset

We load data/metabolomics_dataset.csv if present. Otherwise we simulate: - 1,000 samples, 200 metabolites (some correlated by latent pathways), - two classes (balanced), - mild batch drift on ~10% features, - continuous severity score.

We inject class signal into 20 features and weak signal into 40, the rest noise/correlated redundancy (realistic for LC‚ÄìMS).

def simulate_metabolomics(n=1000, p=200, seed=RANDOM_SEED):
    rng = np.random.default_rng(seed)
    # Latent pathways: 6 blocks with correlated features
    blocks = [40, 40, 30, 30, 30, 30]
    assert sum(blocks) == p
    # Class labels
    y = rng.integers(0, 2, size=n)
    # Base matrix (Gaussian noise)
    X = rng.normal(0, 1, size=(n, p))
    # Add pathway correlation: each block driven by a latent factor
    start = 0
    for b, k in enumerate(blocks):
        z = rng.normal(0, 1, size=(n, 1))  # latent factor per block
        load = rng.normal(0.6, 0.05, size=(1, k))  # moderate correlation
        X[:, start:start+k] += z @ load
        start += k
    # Inject class signal: 20 strong, 40 weak features spread across blocks
    strong_idx = rng.choice(p, size=20, replace=False)
    weak_candidates = np.setdiff1d(np.arange(p), strong_idx)
    weak_idx = rng.choice(weak_candidates, size=40, replace=False)
    effect = (y*2 - 1)[:, None]  # -1 or +1
    X[:, strong_idx] += effect * rng.normal(1.0, 0.1, size=(n, 20))
    X[:, weak_idx]   += effect * rng.normal(0.3, 0.05, size=(n, 40))
    # Batch drift on 10% features
    drift_idx = rng.choice(p, size=int(0.1*p), replace=False)
    drift_trend = np.linspace(-0.8, 0.8, n)[:, None]
    X[:, drift_idx] += drift_trend * rng.normal(0.4, 0.05, size=(1, len(drift_idx)))
    # Severity (continuous outcome) driven by a few PCs + noise
    sev = (0.5*effect.squeeze() + 0.2*rng.normal(0,1,n) + 0.1*X[:, strong_idx].mean(axis=1))
    # Shift to positive space and log-spread (like intensities)
    X = np.exp( X / 2 ) * 1000
    cols = [f'Metabolite_{i+1}' for i in range(p)]
    df = pd.DataFrame(X, columns=cols)
    df['Label'] = y
    df['Severity'] = (sev - sev.min())/(sev.max()-sev.min()) * 10
    return df

Preprocessing & Scaling (Why it matters in metabolomics)

Raw LC‚ÄìMS intensities are typically right-skewed and vary in scale by orders of magnitude. Common choices:

  • Autoscaling (UV): centre to mean 0, scale to unit variance ‚Äî emphasises low-variance features (can amplify noise).
  • Pareto scaling: divide by ‚àöSD ‚Äî middle ground; widely used in metabolomics.
  • Log or log-Pareto: stabilises variance, reduces skew.

Below we provide helpers; you can switch scaling by changing SCALING_MODE.

X = df.filter(like='Metabolite_').astype(float).values
y = df['Label'].astype(int).values
sev = df['Severity'].values

def autoscale(X):
    m = X.mean(axis=0); s = X.std(axis=0, ddof=1); s[s==0]=1
    return (X - m)/s

def pareto(X):
    m = X.mean(axis=0); s = np.sqrt(X.std(axis=0, ddof=1)); s[s==0]=1
    return (X - m)/s

def log_pareto(X):
    Xl = np.log1p(X)
    return pareto(Xl)

SCALING_MODE = 'log_pareto'  # 'autoscale' | 'pareto' | 'log_pareto'
if SCALING_MODE=='autoscale':
    Xs = autoscale(X)
elif SCALING_MODE=='pareto':
    Xs = pareto(X)
else:
    Xs = log_pareto(X)

PCA ‚Äî Core Concepts (Eigenvalues, Eigenvectors, Eigenfactors)

Let (^{np}) be column-centred (after scaling). PCA finds orthonormal directions (eigenvectors) (_k) of the covariance matrix ( = ^T), with eigenvalues (_k) (variance explained by PC(k)).

  • Scores (sample coordinates) on PC(k): (_k = ,_k).
  • Loadings (feature weights) are the eigenvectors (_k).
  • Explained variance of PC(k): (_k / _j _j).
  • Hotelling‚Äôs T¬≤ for sample i: (T^2_i = _{k=1}^a ) over the first (a) PCs.
  • Q-residuals/SPE: squared residual after projecting onto the first (a) PCs: (Q_i = _i - _i ^2).

Eigenfactors ‚Äî what are they?

In chemometrics/metabolomics teaching, Eigenfactors are often used as an intuitive term for the eigen-decomposition factors that drive PCA: the eigenvalues (variance magnitudes) and eigenvectors (directional weights) together. They determine (i) how much each PC matters (eigenvalue), and (ii) how features combine to form that PC (eigenvector). In short: Eigenfactors = the principal axes (loadings) and their importance (eigenvalues). Reporting them correctly (plus cumulative variance) is crucial.

We‚Äôll compute all of these and show scree and broken-stick criteria for choosing PCs.

pca = PCA(n_components=min(Xs.shape[0], Xs.shape[1]), svd_solver='full', random_state=RANDOM_SEED)
scores = pca.fit_transform(Xs)
loadings = pca.components_.T  # p x PCs
eigvals = pca.explained_variance_  # lambda_k
expl_var = pca.explained_variance_ratio_
cum_var = np.cumsum(expl_var)

# Broken-stick model (expected proportion under null of random variance)
p_ = Xs.shape[1]
broken = np.array([sum(1/np.arange(k, p_+1)) / p_ for k in range(1, p_+1)])

plt.figure(figsize=(7,4))

plt.xlabel('PC'); plt.ylabel('Proportion of variance'); plt.title('Scree & Broken-stick (first 10 PCs)'); plt.legend(); plt.tight_layout(); plt.show()

PCA Scores, Hotelling‚Äôs T¬≤ Ellipse, and Q-Residuals

We‚Äôll plot PC1‚ÄìPC2 with a 95% Hotelling‚Äôs T¬≤ ellipse and flag samples with large Q-residuals (outside the model in orthogonal space).

a = 2  # number of PCs to display
T = scores[:, :a]
Lambda = np.diag(eigvals[:a])
T2 = np.sum(T @ la.inv(Lambda) * T, axis=1)  # rowwise

# F-approximation for T¬≤ limit (Jackson, 1991):
n = Xs.shape[0]
alpha = 0.05
T2_lim = (a*(n-1)/(n-a)) * f.ppf(1-alpha, a, n-a)

# Q-residuals (SPE):
P = loadings[:, :a]  # p x a
Xhat = T @ P.T
E = Xs - Xhat
Q = np.sum(E**2, axis=1)
# Heuristic Q limit via percentiles (for teaching); robust methods use Jackson‚Äôs Q-stat or permutation
Q_lim = np.percentile(Q, 95)

# Ellipse for T¬≤ in score space (PC1/PC2)
cov_scores = np.cov(T, rowvar=False)
eigvals_s, eigvecs_s = la.eigh(cov_scores)
order = np.argsort(eigvals_s)[::-1]
eigvals_s, eigvecs_s = eigvals_s[order], eigvecs_s[:, order]
theta = np.linspace(0, 2*np.pi, 200)
radii = np.sqrt(T2_lim) * np.sqrt(eigvals_s)
ellipse = (eigvecs_s @ np.diag(radii) @ np.vstack([np.cos(theta), np.sin(theta)])).T + T.mean(axis=0)

plt.figure(figsize=(7,6))
plt.scatter(T[:,0], T[:,1], c=y, cmap='viridis', alpha=0.7, s=18, label='Samples')
plt.plot(ellipse[:,0], ellipse[:,1], 'k--', lw=1.5, label="95% T¬≤ ellipse")
out_t2 = T2 > T2_lim
out_q  = Q  > Q_lim
plt.scatter(T[out_t2,0], T[out_t2,1], edgecolor='r', facecolor='none', s=80, label='T¬≤ outliers')
plt.scatter(T[out_q,0], T[out_q,1], edgecolor='orange', facecolor='none', s=80, label='Q outliers')
plt.xlabel('PC1'); plt.ylabel('PC2'); plt.title('PCA Scores with T¬≤ & Q outliers'); plt.legend(); plt.tight_layout(); plt.show()

print(f"T¬≤ limit: {T2_lim:.2f}; Q 95th pct: {Q_lim:.2f}; #T¬≤ outliers: {out_t2.sum()}, #Q outliers: {out_q.sum()}")

VIP Scores ‚Äî Variable Importance in Projection

VIP summarises a feature‚Äôs overall contribution to the PLS components (weighted by explained (Y) variance). A common threshold is VIP > 1.

VIP for variable j: [ j = ] where (w{ja}) are weights and (SSY_a) the sum of squares in (Y) explained by component a.

def vip_scores(pls):
    """
    Compute Variable Importance in Projection (VIP) scores
    for scikit-learn PLSRegression.

    Returns
    -------
    vips : (p,) array of VIP scores for X variables in column order.
    """
    T = pls.x_scores_        # (n, A)
    W = pls.x_weights_       # (p, A)
    Q = pls.y_loadings_      # (n_targets, A)

    p = W.shape[0]
    # SSY per component (works for single or multi-target)
    t_ss  = np.sum(T**2, axis=0)        # (A,)
    q_ss  = np.sum(Q**2, axis=0)        # (A,)
    SSY   = t_ss * q_ss                 # (A,)

    # Normalise weights per component
    wnorm2 = np.sum(W**2, axis=0)       # (A,)
    wnorm2 = np.where(wnorm2 == 0, 1e-12, wnorm2)

    # Contribution matrix: (p, A)
    contrib = (W**2 / wnorm2) * SSY     # broadcasts SSY over rows

    denom = SSY.sum()
    denom = denom if denom > 0 else 1e-12

    vips = np.sqrt(p * contrib.sum(axis=1) / denom)  # (p,)
    return vips

vips = vip_scores(pls)
vip_top = (pd.DataFrame({'Feature': Xdf.columns, 'VIP': vips})
           .sort_values('VIP', ascending=False)
           .head(15))
display(vip_top)

plt.figure(figsize=(7,4))

plt.axvline(1, ls='--')     # VIP>1 often treated as important
plt.title('Top VIP features')
plt.tight_layout()
plt.show()

Permutation Test (Essential sanity check)

We permute class labels (break real association), refit PLS-DA, and recompute CV AUC. If the observed AUC is not better than the null distribution, the model likely overfits.

For speed we use 100 permutations (increase to 1,000 for publication-grade).

def cv_auc_pls(X, y, a=2, seed=RANDOM_SEED):
    cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=seed)
    preds = np.zeros_like(y, dtype=float)
    for tr, te in cv.split(X, y):
        m = PLSRegression(n_components=a)
        m.fit(X[tr], y[tr])
        preds[te] = m.predict(X[te]).ravel()
    return roc_auc_score(y, preds)

obs_auc = cv_auc_pls(Xs_pls, y_bin, a=n_comp)
null_aucs = []
for b in range(100):
    yperm = rng.permutation(y_bin)
    null_aucs.append(cv_auc_pls(Xs_pls, yperm, a=n_comp, seed=RANDOM_SEED+b+1))
p_perm = (np.sum(np.array(null_aucs) >= obs_auc) + 1) / (len(null_aucs)+1)
print(f"Observed CV AUC={obs_auc:.3f}; permutation p‚âà{p_perm:.3f}")
plt.figure(figsize=(7,4)); sns.histplot(null_aucs, bins=20, kde=False); plt.axvline(obs_auc, c='red'); plt.title('Permutation null AUCs (PLS-DA)'); plt.xlabel('AUC'); plt.tight_layout(); plt.show()

Bayesian PPCA (PyMC) ‚Äî Uncertainty-aware PCA

Probabilistic PCA (PPCA) assumes (_i = _i + + _i), with (_i(0,)) and (_i(0,^2)). It yields PCs similar to PCA but within a generative model. The Bayesian version gives posterior uncertainty for loadings and noise.

We model 2 latent dimensions for illustration (adjust as needed). We‚Äôll use a subset of 400 samples √ó 80 features for speed during teaching.

sub_n, sub_p = 400, 80
idx = rng.choice(Xs.shape[0], size=sub_n, replace=False)
jdx = rng.choice(Xs.shape[1], size=sub_p, replace=False)
Xpp = Xs[np.ix_(idx, jdx)]

A = 2  # latent dims
with pm.Model() as ppca:
    mu = pm.Normal('mu', mu=0, sigma=1, shape=sub_p)
    W = pm.Normal('W', mu=0, sigma=1, shape=(sub_p, A))
    z = pm.Normal('z', mu=0, sigma=1, shape=(sub_n, A))
    sigma = pm.HalfNormal('sigma', sigma=1)
    Xhat = mu + pm.math.dot(z, W.T)
    pm.Normal('X', mu=Xhat, sigma=sigma, observed=Xpp)
    trace_ppca = pm.sample(1000, tune=1000, target_accept=0.9, random_seed=RANDOM_SEED, return_inferencedata=True)

az.plot_forest(trace_ppca, var_names=['sigma'], combined=True); plt.show()
print(az.summary(trace_ppca, var_names=['sigma'], hdi_prob=0.95))

Random Forest (ML add-on) with Permutation Importance

We add a non-linear classifier and compute permutation importance, which is less biased towards high-cardinality features than Gini importance. We use nested CV for honest performance.

Xrf = Xs
yrf = y_bin
outer = StratifiedKFold(n_splits=5, shuffle=True, random_state=RANDOM_SEED)
inner = StratifiedKFold(n_splits=3, shuffle=True, random_state=RANDOM_SEED)
rf = RandomForestClassifier(random_state=RANDOM_SEED)
grid = {'n_estimators':[200,400], 'max_features':['sqrt','log2'], 'max_depth':[None, 8]}
clf = GridSearchCV(rf, grid, scoring='roc_auc', cv=inner, n_jobs=-1)
auc_outer = []
for tr, te in outer.split(Xrf, yrf):
    clf.fit(Xrf[tr], yrf[tr])
    proba = clf.predict_proba(Xrf[te])[:,1]
    auc_outer.append(roc_auc_score(yrf[te], proba))
print(f"Nested CV ROC AUC (mean¬±SD): {np.mean(auc_outer):.3f} ¬± {np.std(auc_outer):.3f}")

# Permutation importance on full data with best model (illustrative)

best.fit(Xrf, yrf)
perm = permutation_importance(best, Xrf, yrf, n_repeats=20, random_state=RANDOM_SEED)
